RV32I instruction walk program
==============================

Purpose
-------
This program is meant for waveform exploration, not for pass/fail checking.
It executes a broad set of supported RV32I instructions so the processor can
be observed step by step in simulation.

Startup note
------------
This ROM starts directly at `_start` and does not rely on `crt0.S`. That keeps
the program small enough to fit the current 64-word instruction ROM used by the
processor wrapper.

Program flow
------------
1. Execute one FENCE instruction.
2. Load integer values into registers with immediate instructions.
3. Execute the R-type ALU operations.
4. Execute the I-type ALU operations.
5. Exercise byte, halfword and word stores/loads in data memory.
6. Exercise all implemented branch types with taken branches.
7. Execute JAL and then return with JALR.
8. Halt the core with ECALL.
9. EBREAK is also encoded after ECALL, but it is not reached in normal
   execution because this core halts the PC on the first SYSTEM halt
   instruction.

Data-memory addresses used
--------------------------
- Base address: 0x00000080
- Byte test:    0x00000080
- Halfword:     0x00000082
- Word:         0x00000084
- Final store:  0x00000088

Instruction meanings
--------------------
add    : rd = rs1 + rs2
sub    : rd = rs1 - rs2
sll    : rd = rs1 logical-left-shifted by rs2[4:0]
slt    : rd = 1 when rs1 < rs2 using signed comparison, else 0
sltu   : rd = 1 when rs1 < rs2 using unsigned comparison, else 0
xor    : rd = rs1 XOR rs2
srl    : rd = rs1 logical-right-shifted by rs2[4:0]
sra    : rd = rs1 arithmetic-right-shifted by rs2[4:0]
or     : rd = rs1 OR rs2
and    : rd = rs1 AND rs2

addi   : rd = rs1 + immediate
slti   : rd = 1 when rs1 < immediate using signed comparison, else 0
sltiu  : rd = 1 when rs1 < immediate using unsigned comparison, else 0
xori   : rd = rs1 XOR immediate
ori    : rd = rs1 OR immediate
andi   : rd = rs1 AND immediate
slli   : rd = rs1 logical-left-shifted by shamt
srli   : rd = rs1 logical-right-shifted by shamt
srai   : rd = rs1 arithmetic-right-shifted by shamt

lb     : load signed byte from memory
lh     : load signed halfword from memory
lw     : load word from memory
lbu    : load unsigned byte from memory
lhu    : load unsigned halfword from memory

sb     : store low byte of rs2 into memory
sh     : store low halfword of rs2 into memory
sw     : store word of rs2 into memory

beq    : branch when rs1 == rs2
bne    : branch when rs1 /= rs2
blt    : branch when rs1 < rs2 using signed comparison
bge    : branch when rs1 >= rs2 using signed comparison
bltu   : branch when rs1 < rs2 using unsigned comparison
bgeu   : branch when rs1 >= rs2 using unsigned comparison

lui    : load upper immediate into rd
auipc  : rd = current PC + upper immediate
jal    : jump to target and write return address into rd
jalr   : jump to register target and optionally write return address

fence  : ordering instruction; this core currently treats it as a NOP
ecall  : system call instruction; this core halts the PC on it
ebreak : breakpoint instruction; this core also halts the PC on it

Execution note
--------------
In this implementation, ECALL and EBREAK are both supported as halt
instructions. Because the PC stops on the first one reached, a single run
cannot execute both of them. This ROM executes ECALL and leaves EBREAK as an
encoded but unreachable final instruction.
